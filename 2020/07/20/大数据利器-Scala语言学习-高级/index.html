Scala高级

一、集合

Scala 的集合有三大类：序列 Seq、集 Set、映射 Map，所有的集合都扩展自 Iterable 特质，在 Scala 中集合有可变（mutable）和不可变（immutable）两种类型。

  • 1.Set、Map 是 Java 中也有的集合。
  • 2.Seq 是 Java 没有的，我们发现 List 归属到 Seq 了,因此这里的 List 就和 java 不是同一个概念了。
  • 3.我们前面的 for 循环有一个 1 to 3 ,就是 IndexedSeq 下的 Vector。
  • 4.String 也是属于 IndexeSeq。
  • 5.我们发现经典的数据结构比如 Queue 和 Stack 被归属到 LinearSeq。
  • 6.大家注意 Scala 中的 Map 体系有一个 SortedMap,说明 Scala 的 Map 可以支持排序。
  • 7.IndexSeq 和 LinearSeq 的区别[IndexSeq 是通过索引来查找和定位，因此速度快，比如 String 就是一个索引集合，通过索引即可定位] [LineaSeq 是线型的，即有头尾的概念，这种数据结构一般是通过遍历来查找，它的价值在于应用到一些。具体的应用场景 (电商网站, 大数据推荐系统 :最近浏览的 10 个商品)]

4、队列

    1. 队列是一个有序列表，在底层可以用数组或是链表来实现。
    1. 其输入和输出要遵循先入先出的原则。即：先存入队列的数据，要先取出。后存入的要后取出。
    1. 在 Scala 中，由设计者直接给我们提供队列类型使用。
    1. 在 scala 中, 有 scala.collection.mutable.Queue 和 scala.collection.immutable.Queue , 一般来说，我们在开发中通常使用可变集合中的队列

5、映射–Map

    1. Scala 中，有可变 Map (scala.collection.mutable.Map) 和 不可变 Map(scala.collection.immutable.Map)。

二、模式匹配

    1. 如果所有 case 都不匹配，那么会执行 case _ 分支，类似于 Java 中 default 语句
    1. 如果所有 case 都不匹配，又没有写 case _ 分支，那么会抛出 MatchError
    1. 每个 case 中，不用 break 语句，自动中断 case
    1. 可以在 match 中使用其它类型，而不仅仅是字符,可以是表达式
    1. => 等价于 java swtich 的 :
    1. => 后面的代码块到下一个 case， 是作为一个整体执行，可以使用{} 扩起来，也可以不扩。

5、对象匹配

    1. 构建对象时 apply 会被调用 ，比如 val n1 = Square(5)
    1. 当将 Square(n) 写在 case 后时[case Square(n) => xxx]，会默认调用 unapply 方法(对象提取器)
    1. number 会被 传递给 def unapply(z: Double) 的 z 形参
    1. 如果返回的是 Some 集合，则 unapply 提取器返回的结果会返回给 n 这个形参
    1. case 中对象的 unapply 方法(提取器)返回 some 集合则为匹配成功
    1. 返回 None 集合则为匹配失败

6、样例类

    1. 样例类仍然是类。
    1. 样例类用 case 关键字进行声明。
    1. 样例类是为模式匹配(对象)而优化的类。
    1. 构造器中的每一个参数都成为 val——除非它被显式地声明为 var（不建议这样做）。
    1. 在样例类对应的伴生对象中提供 apply 方法让你不用 new 关键字就能构造出相应的对象。
    1. 提供 unapply 方法让模式匹配可以工作。
    1. 将自动生成 toString、equals、hashCode 和 copy 方法(有点类似模板类，直接给生成，供程序员使用)。
    1. 除上述外，样例类和其他类完全一样。你可以添加方法和字段，扩展它们。

7、密封类

    1. 如果想让 case 类的所有子类都必须在申明该类的相同的源文件中定义，可以将样例类的通用超类声明为sealed，这个超类称之为密封类。
    1. 密封就是不能在其他文件中定义/使用类。

三、函数式编程

1、偏函数

在对符合某个条件，而不是所有情况 进行逻辑操作时，使用偏函数是一个不错的选择将包在大括号内的一组 case 语句封装为函数，我们称之为偏函数，它只对会作用于指定类型的参数或指定范围值的参数实施计算，超出范围的值会忽略.

偏函数在 Scala 中是一个特质 PartialFunction

    1. 使用构建特质的实现类(使用的方式是 PartialFunction 的匿名子类)
    1. PartialFunction 是个特质(看源码)
    1. 构建偏函数时，参数形式 [Any, Int]是泛型，第一个表示传入参数类型，第二个表示返回参数
    1. 当使用偏函数时，会遍历集合的所有元素，编译器执行流程时先执行 isDefinedAt()如果为 true ,就会执行 apply,构建一个新的 Int 对象返回
    1. 执行 isDefinedAt() 为 false 就过滤掉这个元素，即不构建新的 Int 对象.
    1. map 函数不支持偏函数，因为 map 底层的机制就是所有循环遍历，无法过滤处理原来集合的元素
    1. collect 函数支持偏函数

没有名字的函数就是匿名函数，可以通过 函数表达式，来设置匿名函数

能够接受函数作为参数的函数，叫做高阶函数 (higher-order function)。可使应用程序更加健壮。 高阶函数可以返回一个匿名函数。

4、闭包

基本介绍：闭包就是一个函数和与其相关的引用环境（变量/值）组合的一个整体(实体)。

f就是一个闭包

5、函数柯里化(curry)

    1. 函数编程中，接受多个参数的函数都可以转化为接受单个参数的函数，这个转化过程就叫柯里化。
    1. 柯里化就是证明了函数只需要一个参数而已。其实我们刚才的学习过程中，已经涉及到了柯里化操作。
    1. 不用设立柯里化存在的意义这样的命题。柯里化就是以函数为主体这种思想发展的必然产生的结果。(即：柯里化是面向函数思想的必然产生结果)

2、协变、逆变和不变

Scala 的协变(+)，逆变(-)，协变 covariant、逆变 contravariant、不可变 invariant

    1. C[+T]：如果 A 是 B 的子类，那么 C[A]是 C[B]的子类，称为协变。
    1. C[-T]：如果 A 是 B 的子类，那么 C[B]是 C[A]的子类，称为逆变。
    1. C[T]：无论 A 和 B 是什么关系，C[A]和 C[B]没有从属关系。称为不变。
